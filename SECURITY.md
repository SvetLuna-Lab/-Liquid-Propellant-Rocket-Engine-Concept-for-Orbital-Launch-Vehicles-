# Security Policy for Liquid-Propellant-Rocket-Engine-Concept-for-Orbital-Launch-Vehicles

## Reporting a Vulnerability  
If you discover a security vulnerability in this project, please send an email to **[your-security-email@example.com]** with the subject line:  
SECURITY REPORT — Liquid-Propellant-Rocket-Engine-Concept-for-Orbital-Launch-Vehicles

markdown

Include:  
- A clear description of the issue  
- Steps to reproduce if applicable  
- Version of the project where you found it  
- Your contact information (optional if you wish to remain anonymous)  
We will acknowledge your report within **7 calendar days**, and aim to provide a fix within a reasonable timeframe.

## Supported Versions  
We support the **main branch** and any release tagged `vX.Y.Z` where X.Y.Z is the latest stable release.  
Older versions may not receive security patches.

## Disclosure Policy  
We follow a coordinated disclosure process:  
1. We privately receive your vulnerability report.  
2. We analyse and prepare a fix–patch or mitigation.  
3. We notify you when a public advisory is ready.  
4. We publish the advisory and fix simultaneously so users can apply the update.

## Security Best Practices for Users  
- Use the latest release of the project.  
- Review dependency versions and update regularly.  
- Do **not** commit secrets (passwords, keys) into this project’s repository.  
- Run analyses (e.g., static code analysis, dependency vulnerability scanning) when integrating into your system.

## Known Issues  
As of this writing, we are not aware of any major unpatched vulnerabilities.  
If you detect one, please report it through the process above.

## Third-Party Dependencies  
This project uses open-source libraries (e.g., numpy, matplotlib, PyYAML). Up-to-date scanning for vulnerable dependencies is recommended.
