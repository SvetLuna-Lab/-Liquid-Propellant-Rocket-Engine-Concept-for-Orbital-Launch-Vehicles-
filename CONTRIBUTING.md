## How can you contribute?  
You can contribute in multiple ways:  
- Reporting bugs or incorrect behaviour  
- Suggesting improvements or new features  
- Writing or improving documentation (docs/)  
- Enhancing scripts and computations (calculations/, scripts/)  
- Submitting pull requests for changes  

## Getting started  
1. Fork the repository.  
2. Clone your fork locally:  
   ```bash  
   git clone https://github.com/your-username/your-repo.git  
   cd your-repo  
Create a feature branch:

bash

git checkout -b feature/your-feature-name  
Make your changes, commit them:

bash

git commit -m "feat: short description of your change"  
Push to your fork:

bash

git push origin feature/your-feature-name  
Open a Pull Request and explain your changes clearly.

Code style & commit messages
Use clear, descriptive commit messages (e.g., feat: add engine sizing script, fix: correct unit conversion).

Keep changes focused in one pull request.

Ensure documentation is updated when code or configuration changes.

Pull Request review
We will review your PR and may request changes or ask for clarification.

Be open to feedback, and welcome improvements.

Once approved, your contribution will be merged.

Reporting bugs
Use the issue tracker:

Provide steps to reproduce the issue.

Environment: OS, Python version, dependencies.

Expected vs. actual behaviour.
